Your boss hands you a set of "Inputs" and "Outputs" and asks you if you can design the circuitry for this set of data. You have at your desposal the "basic" Gates: AND, OR, NOT, XOR, NAND, NOR. If there are only a few Inputs and one Output you may intuitively play around with the Gates and come up with a Solution. There's are better ways. One is called "Boolean Algebra".

Around the middle of the 19th century, a mathematician, George Boole, wondered whether or not "mathematics could be applied to how people think?". He created a new field of mathematics called "Boolean Logic" and then, somewhat latter, "Boolean Algebra". It wasn't until 1937 that Claude Shannon in his 1937 master's thesis made the connection between "Boolean Algebra" and "Electronic Relays"(Electrical Switches which are binary in nature). In 1945 the first "programmable computer", ENIAC (Electronic Numerical Integrator And Computer) was built utilizing Gates(AND, OR, NOT ie Boolean Algebra).

It is this "Boolean Algebra" that provides us with a systematic tool for coming up with a solution to our problem. We will proceed in the following order:

  1. Place Inputs and Outputs into a Truth Table
  2. Use some characteristics of the Truth Table as an aid to make the conversion into Boolean Algebraic expressions and/or equations
  3. Simplify/Solve them for your final "design circuit"

Truth Tables and Boolean Algebra are two different entities. First consider Truth Tables. In general:

  1. Table Heads only contain "Labels"(English words and/or symbols ie "Inputs", "P0"), "Output" etc.
  2. Table Bodies only contain Binary Values ie ("0","1").

Now consider Boolean Algebra. It consists of:

  1. "Boolean Variables"(eg P1, P0)
  2. "Boolean Operators"(eg AND, OR, NOT).

"Boolean Variables" can be viewed as a combination of: A Label, A Container, and Contents of that Container, which can be "changed" over time.

NOTE: The Label(eg P0) in the Truth Table can also be thought of as a container(eg its contents is the Column below it in the Table Body. This Content, however, is fixed by definition and cannot be changed: thus it's not a variable!) Important: P0 in the Truth Table is NOT the same as the Boolean variable P0.

We will now make begin the translation. Keep in mind the fundamentals of Boolean Logic and Boolean Algebra: "AND", "OR", "NOT" along with the English Words: "and", "or", "not". Would you be surprized if there is a close connection between them.

It only has One Input (P0). It's BINARY so it can only take on 2 values: "0" or "1". Can P0 be "0" and "1" AT THE SAME TIME?

NO! Binary is like your light switch at home. It can be "UP"(light on) or "DOWN"(light off) but not both at the same time!

Each row of the Truth Table, therefore, represets a unique "state". Thus we say the NOT Gate has two "states", where each row represents a particular "state".

Let's call row 1 "State 1(S1)" and row 2 "State 2(S2)". Hopefully the following "English" sentence makes sense:

The NOT Gate is either in "State 1"(row 1) or "State 2"(row 2).

The key word in that sentence is the English word "or".

Consider the following "English" sentence where "or" is replaced with "and":

The NOT Gate is either in "State 1"(row 1) and "State 2"(row 2).
Does that sentence make logical sense? The two words: ("either", "and") don't go together but ("either", "or") do!

Step 3: Convert the English sentences into Algebraic equations.
We now have the two English sentences for the Truth Table above:

  1. [(P1 = 0) and (P0 = 0)] or [(P1 = 0) and (P0 = 1)] or [(P1 = 1) and (B0 = 0)] equals 0
  2. [(P1 = 1) and (P0 = 1)] equals 1

We need to convert these into an Boolean Algebraic Equation! But how? First consider the following:
For the English Expressions we have:

  1. labels: P1, P0 which are English Labels(NOT Variables)
  2. numeric values: 0, 1
  3. words: "and", "or", "equals"

For Boolean Algebra we have:

  1. variable: such as P1, P0
  2. numeric values(operands): 0, 1
  3. Logic Gates(operators): "AND", "OR", "="

Replace "and" with "AND", "or" with "OR", "equals" with "=". The Output values are fixed values in both cases: 0 or 1. Nothing needs to be done!

As was mentioned previously, the labels P1 and P0 of the Truth Table cannot be directly replaced with the labels for the corresponding Algebraic Variables. The following is not obvious but after you think about it, it does make sense. We'll use P0 as an example.

P0 is a Variable which can have a Value of "0" or "1". Consider NOT P0. It also can have a Value of "0" or "1". The NOT operator, however, reverses the Value in P0 (eg if P0) = 0 then NOT P0 = 1. If P0 = 1 then NOT P0 = 0. Thus we can two options:

  • Replace "0" with P0
  • and replace "1" with NOT P0
or
  • Replace "1" with P0
  • and replace "0" with NOT P0
While both options will work, it turns out the 2nd option is preferable.

For State 2(S2) the Input Values for P1 and P0 are "0", "1" or "P1", "NOT P0" where the first pair P1, P0 are Truth Table Lables and the latter pair "P1", "NOT P0 are Boolean Variables!

State 2(S2) can now be expressed as: P1 AND "NOT P0, which is a Boolean Expression where "P1", P0 are Boolean Variables and "AND", "NOT" are Boolean Operators.

Repeat this for States 1,3,4

For State 1(S1): (NOT P1) AND (NOT P0)
For State 3(S3): (P1) AND (NOT P0)
For State 4(S4): (P1) AND (P0)

For Q0 = "0": (P1) AND (P0) = 1
